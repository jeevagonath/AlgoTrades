### PAGE 3 ###

 
 
 
Login and User Details 
 
 
 
 
 
 
 
1. Redirect the user to the login page.  
2. After successful log in, get the Code from the redirect URL.  
3. POST the code and checksum (SHA-256 of Client_id + secret code + code) to call the /GenAcsTok api 
request. 
4. Get the access_token in response.  
5. Include the access_token in the Authorization header when making each individual API call. 
(Postman)

Request Details : 
Parameter Name  Possible value  Description 
jData*    Should send json object with fields in below list 
 
Json Fields  Possible value  Description 
code*    code will get from successful login url 
checksum*    It is a SHA-256 hash created from the combination of the Client id, 
secret key, and code without any spaces. 
             Example: 
If your client id is ABC, secret code is 123, and code is 
x1y2z3, Combine them as: ABC123x1y2z3
 Then generate the SHA-256 hash of that string. 
 
 
Example:  
curl --location 'http://apitest.kambala.co.in/NorenWClientAPI/GenAcsTok' \ 
--d 'jData={\"code\":\"c61de38b-7c0c-46e4-8abe-
acb1f6a15089\",\"checksum\":\"e5ff9e0357cfa219885495ace4390b38280a39790ad434ea713afd27e6ec93
c5\"}'

Response data will be in json format with below fields.

3.NetSellQty = daysellqty + cfsellqty; 
 
4.ActualSellAvgPrice  
        if (NetSellQty != 0) { 
              ActualSellAvgPrice =  ((daysellamt / mult) +(upldprc * prcftr *  cfsellqty)) /NetSellQty;

        if (NetBuyQty != 0) {

ActualSellAvgPrice =  ((daysellamt / mult) +(upldprc * prcftr *  cfsellqty)) /(NetSellQty*prcftr); 
ActualBuyAvgPrice =  ((daybuyamt / mult) +(upldprc * prcftr * cfbuyqty)) /(NetBuyQty*prcftr);]
